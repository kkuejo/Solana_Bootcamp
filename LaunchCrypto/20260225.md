# Solana開発環境のセットアップとAirdropプロジェクト

**日付**: 2026年2月25日
**学習内容**: Solana開発環境の構築、@solana/kitを使ったAirdropプログラムの作成

## 1. Solanaの概要

### 1.1 Solanaとは

Solanaは、高速・低コストなトランザクション処理を実現するブロックチェーンプラットフォームです。以下の特徴があります：

- **高速**: 理論上最大65,000 TPS（Transactions Per Second）
- **低コスト**: トランザクション手数料が非常に安価
- **Proof of History (PoH)**: 独自のコンセンサスメカニズムによる高速処理
- **スマートコントラクト**: Rust、C、C++でプログラム（スマートコントラクト）を開発可能

### 1.2 Solanaのネットワーク

Solanaには3つのネットワークとローカル開発環境があります：

| ネットワーク | 用途 | HTTP URL | WebSocket URL |
|-------------|------|----------|---------------|
| **Mainnet Beta** | 本番環境 | `https://api.mainnet-beta.solana.com` | `wss://api.mainnet-beta.solana.com` |
| **Devnet** | 開発・テスト用 | `https://api.devnet.solana.com` | `wss://api.devnet.solana.com` |
| **Testnet** | バリデーターテスト用 | `https://api.testnet.solana.com` | `wss://api.testnet.solana.com` |
| **Localhost** | ローカル開発用 | `http://127.0.0.1:8899` | `ws://127.0.0.1:8900` |

### 1.3 主要な用語

- **SOL**: Solanaのネイティブトークン
- **Lamport**: SOLの最小単位（1 SOL = 1,000,000,000 Lamports）
- **Keypair**: 公開鍵と秘密鍵のペア（ウォレットアドレスの生成に使用）
- **Airdrop**: テスト用SOLを無料で受け取る仕組み（devnet/testnet/localhostで利用可能）

### 1.4 @solana/kitについて

`@solana/kit`は、旧`@solana/web3.js`に代わる新しいSolana JavaScript/TypeScript SDKです。Anza社が開発しており、以下の特徴があります：

| 特徴 | 旧 @solana/web3.js | 新 @solana/kit |
|------|-------------------|----------------|
| スタイル | クラスベース | 関数型プログラミング |
| モジュール | CommonJS | ESM（ES Modules）推奨 |
| 数値型 | `number` | `bigint` |
| 暗号処理 | 同期 | 非同期（Web Crypto API） |
| ツリーシェイキング | 不可 | 可能（未使用コードの除去） |

### 2.4 package.jsonの設定

`@solana/kit`はESM（ES Modules）を推奨しているため、`package.json`の`type`を`module`に設定します：

```json
{
  "name": "airdropping",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [],
  "author": "",
  "license": "ISC",
  "type": "module",
  "dependencies": {
    "@solana/kit": "^6.1.0"
  }
}
```

**ポイント:**
- `"type": "module"` — ESM（`import`/`export`構文）を使用するための設定。旧`@solana/web3.js`では`"commonjs"`（`require()`構文）だったが、`@solana/kit`ではESMが推奨
- `@solana/kit` — 旧`@solana/web3.js`に代わる新しいSolana JavaScript SDK

## 3. Airdropプログラムの作成

### 3.1 index.jsの実装

```javascript
import {
    createSolanaRpc,
    createSolanaRpcSubscriptions,
    generateKeyPairSigner,
    airdropFactory,
    lamports,
} from "@solana/kit";

const LAMPORTS_PER_SOL = BigInt(1_000_000_000);

const RPC_URL = "http://127.0.0.1:8899";
const WS_URL = "ws://127.0.0.1:8900";

const rpc = createSolanaRpc(RPC_URL);
const rpcSubscriptions = createSolanaRpcSubscriptions(WS_URL);

const signer = await generateKeyPairSigner();

const getWalletBalance = async () => {
    try {
        const { value: balance } = await rpc.getBalance(signer.address).send();
        console.log(`Wallet: ${signer.address}`);
        console.log(`Wallet Balance is ${Number(balance) / Number(LAMPORTS_PER_SOL)} SOL`);
    } catch (error) {
        console.error(error);
    }
};

const airDropSol = async () => {
    try {
        const airdrop = airdropFactory({ rpc, rpcSubscriptions });
        await airdrop({
            commitment: "confirmed",
            recipientAddress: signer.address,
            lamports: lamports(LAMPORTS_PER_SOL),
        });
        console.log("Airdrop success!");
    } catch (error) {
        console.error("Airdrop failed:", error.message);
    }
};

const main = async () => {
    await getWalletBalance();
    await airDropSol();
    await getWalletBalance();
};

main();
```

### 3.2 コードの詳細解説

#### 3.2.1 インポート

```javascript
import {
    createSolanaRpc,
    createSolanaRpcSubscriptions,
    generateKeyPairSigner,
    airdropFactory,
    lamports,
} from "@solana/kit";
```

| モジュール | 説明 |
|-----------|------|
| `createSolanaRpc` | HTTP RPCクライアントを作成する関数（残高取得、Airdropリクエストなど） |
| `createSolanaRpcSubscriptions` | WebSocket RPCクライアントを作成する関数（トランザクション確認の待機に使用） |
| `generateKeyPairSigner` | 新しいウォレット（公開鍵+秘密鍵のペア）を生成する関数 |
| `airdropFactory` | Airdrop機能を作成するファクトリー関数 |
| `lamports` | Lamport単位の値を型付けするラッパー関数 |

**旧@solana/web3.jsとの比較:**

| 旧 @solana/web3.js | 新 @solana/kit | 説明 |
|-------------------|----------------|------|
| `Connection` | `createSolanaRpc` + `createSolanaRpcSubscriptions` | 1つのクラスが2つの関数に分離 |
| `Keypair` | `generateKeyPairSigner` | クラスから関数へ |
| `LAMPORTS_PER_SOL` | 自分で定義: `BigInt(1_000_000_000)` | 定数がエクスポートされなくなった |
| `clusterApiUrl("devnet")` | URL文字列を直接指定 | ヘルパー関数が廃止 |

#### 3.2.2 定数とRPC接続の設定

```javascript
const LAMPORTS_PER_SOL = BigInt(1_000_000_000);

const RPC_URL = "http://127.0.0.1:8899";
const WS_URL = "ws://127.0.0.1:8900";

const rpc = createSolanaRpc(RPC_URL);
const rpcSubscriptions = createSolanaRpcSubscriptions(WS_URL);
```

- `LAMPORTS_PER_SOL` — `@solana/kit`ではこの定数がエクスポートされないため、自分で定義する。`BigInt`を使用するのは、`@solana/kit`が全ての金額を`bigint`型で扱うため
- `createSolanaRpc(RPC_URL)` — HTTP RPCクライアント。残高取得やAirdropリクエストなどの通常のRPC呼び出しに使用
- `createSolanaRpcSubscriptions(WS_URL)` — WebSocket RPCクライアント。トランザクションの確認待機（サブスクリプション）に使用

旧`@solana/web3.js`では`Connection`クラス1つで両方の役割を担っていたが、`@solana/kit`ではHTTP（リクエスト/レスポンス）とWebSocket（サブスクリプション）が明確に分離されている。

#### 3.2.3 ウォレットの生成

```javascript
const signer = await generateKeyPairSigner();
```

- `generateKeyPairSigner()` — 新しいウォレットを生成する。Web Crypto APIを使用するため`await`が必要（旧`Keypair.generate()`は同期的だった）
- `signer.address` — ウォレットのアドレス（Base58エンコード済みの文字列）。旧`keypair.publicKey.toBase58()`に相当

**注意**: プログラム実行のたびに新しいウォレットが生成されるため、毎回異なるアドレスになります。

#### 3.2.4 残高取得関数 — getWalletBalance

```javascript
const getWalletBalance = async () => {
    try {
        const { value: balance } = await rpc.getBalance(signer.address).send();
        console.log(`Wallet: ${signer.address}`);
        console.log(`Wallet Balance is ${Number(balance) / Number(LAMPORTS_PER_SOL)} SOL`);
    } catch (error) {
        console.error(error);
    }
};
```

**処理の流れ:**
1. `rpc.getBalance(signer.address)` — 指定したアドレスの残高を取得するRPCリクエストを作成
2. `.send()` — RPCリクエストを実際に送信する。`@solana/kit`では全てのRPC呼び出しに`.send()`が必要
3. `{ value: balance }` — 結果は`{ value: bigint }`の形式で返されるため、分割代入で取り出す
4. `Number(balance) / Number(LAMPORTS_PER_SOL)` — `bigint`を`number`に変換してSOL単位で表示

#### 3.2.5 Airdrop関数 — airDropSol

```javascript
const airDropSol = async () => {
    try {
        const airdrop = airdropFactory({ rpc, rpcSubscriptions });
        await airdrop({
            commitment: "confirmed",
            recipientAddress: signer.address,
            lamports: lamports(LAMPORTS_PER_SOL),
        });
        console.log("Airdrop success!");
    } catch (error) {
        console.error("Airdrop failed:", error.message);
    }
};
```

**処理の流れ:**
1. `airdropFactory({ rpc, rpcSubscriptions })` — Airdrop関数を作成する。HTTP RPC（リクエスト送信用）とWebSocket RPC（確認待機用）の両方を渡す
2. `airdrop({ ... })` — Airdropを実行し、トランザクションが確認されるまで自動的に待機する
3. `lamports(LAMPORTS_PER_SOL)` — 1 SOL分のLamportsを型付けしてAirdropをリクエスト

**airdropのパラメータ:**

| パラメータ | 説明 |
|-----------|------|
| `commitment` | トランザクション確認レベル（`"confirmed"`で十分な確認を待機） |
| `recipientAddress` | Airdropの受取先アドレス |
| `lamports` | リクエストする金額（Lamport単位、`lamports()`でラップ） |

旧`@solana/web3.js`では`requestAirdrop`→`getLatestBlockhash`→`confirmTransaction`と3ステップ必要だったが、`@solana/kit`の`airdropFactory`はAirdropリクエストからトランザクション確認まで1ステップで完結する。

#### 3.2.6 メイン関数

```javascript
const main = async () => {
    await getWalletBalance();  // Airdrop前の残高を表示
    await airDropSol();         // Airdropを実行
    await getWalletBalance();  // Airdrop後の残高を表示
};

main();
```

Airdrop前後の残高を表示することで、SOLが正しく受け取れたことを確認できます。

## 4. プログラムの実行

### 4.1 ローカルバリデーターの起動

Airdropをローカル環境で実行するには、まずSolana CLIに含まれる`solana-test-validator`コマンドでローカルバリデーターを起動します。

```bash
# ターミナル1: ローカルバリデーターを起動
solana-test-validator
```

**期待される出力:**
```
Ledger location: test-ledger
Log: test-ledger/validator.log
⠁ Initializing...
Identity: XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
Genesis Hash: XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
Version: X.XX.XX
Shred Version: XXXXX
Gossip Address: 127.0.0.1:1024
TPU Address: 127.0.0.1:1027
JSON RPC URL: http://127.0.0.1:8899
⠁ 00:00:20 | Processed Slot: 40 | Confirmed Slot: 40 | ...
```

このコマンドにより、ローカルマシン上にSolanaネットワークが立ち上がり、`http://127.0.0.1:8899`（HTTP）と`ws://127.0.0.1:8900`（WebSocket）でアクセスできるようになります。

### 4.2 プログラムの実行

```bash
# ターミナル2: Airdropプログラムを実行
node index.js
```

**期待される出力:**
```
Wallet: FMXz95TY8z6cGjX83zMaffjjNvup2ts9ojLr5D5UopM5
Wallet Balance is 0 SOL
Airdrop success!
Wallet: FMXz95TY8z6cGjX83zMaffjjNvup2ts9ojLr5D5UopM5
Wallet Balance is 1 SOL
```

### 4.3 devnetでの実行について

ローカルバリデーターの代わりにdevnetを使用することも可能です。その場合、RPC URLを変更します：

```javascript
const RPC_URL = "https://api.devnet.solana.com";
const WS_URL = "wss://api.devnet.solana.com";
```

**devnetの制限事項:**
- 1日あたり/IPあたりのAirdropリクエスト数に制限がある（429 Too Many Requests）
- faucetサーバーが不安定な場合がある（Internal error）
- 代替手段として https://faucet.solana.com からWeb経由でSOLを受け取れる

**ローカルバリデーターのメリット:**
- レート制限なし
- ネットワーク遅延なし
- オフラインでも動作
- 何度でもAirdrop可能

## 5. トラブルシューティング

### 5.1 よくあるエラーと解決方法

**エラー1: Connection refused（ローカルバリデーター未起動）**
```
Error: connect ECONNREFUSED 127.0.0.1:8899
```
**解決**: 別のターミナルで`solana-test-validator`を起動してから再実行する

**エラー2: 429 Too Many Requests（devnet使用時）**
```
Airdrop failed: 429 Too Many Requests
```
**解決**: devnetのレート制限に達している。時間を置くか、 https://faucet.solana.com を使用する。またはローカルバリデーターを使用する

**エラー3: Internal error（devnet/testnet使用時）**
```
Airdrop failed: airdrop to ... failed: Internal error
```
**解決**: faucetサーバー側の問題。時間を置いて再試行するか、ローカルバリデーターを使用する

**エラー4: require is not defined（モジュール形式の不一致）**
```
ReferenceError: require is not defined in ES module scope
```
**解決**: `package.json`の`"type"`が`"module"`になっていることを確認する。`@solana/kit`はESM（`import`構文）を推奨しており、CommonJS（`require`構文）ではない

## 6. 学習の成果

### 6.1 習得したスキル

1. **Solana CLIのインストール**: Solana CLIツールセットのインストールと、`solana-test-validator`によるローカルバリデーターの起動
2. **Node.jsプロジェクトの初期化**: `npm init`と`@solana/kit`のインストール、ESMの設定
3. **ウォレットの生成**: `generateKeyPairSigner()`を使った公開鍵・秘密鍵ペアの生成
4. **ネットワーク接続**: `createSolanaRpc`と`createSolanaRpcSubscriptions`によるSolanaネットワークへの接続
5. **残高取得**: `rpc.getBalance().send()`による残高の取得とLamport→SOL変換
6. **Airdropの実行**: `airdropFactory`によるテスト用SOLの取得とトランザクション確認
7. **新SDKの理解**: 旧`@solana/web3.js`と新`@solana/kit`のAPI差異の理解

### 6.2 重要な学び

- **ローカルバリデーターの活用**: devnetのレート制限を回避し、安定した開発環境を確保できる。ローカルバリデーターはSolana CLIに含まれる`solana-test-validator`コマンドで起動する
- **新SDKの関数型スタイル**: `@solana/kit`はクラスベースではなく関数型のAPI設計。`Connection`→`createSolanaRpc`、`Keypair`→`generateKeyPairSigner`など
- **HTTP/WebSocketの分離**: 旧SDKでは`Connection`1つで済んでいた接続が、`rpc`（HTTP）と`rpcSubscriptions`（WebSocket）に分離された
- **bigintの使用**: 金額はすべて`bigint`型で扱い、`lamports()`関数でラップする
- **非同期処理**: Solanaとの通信だけでなく、鍵ペア生成も非同期（`await`が必要）
